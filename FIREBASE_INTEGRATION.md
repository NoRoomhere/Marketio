## Как это работает:

### Регистрация:
- Пользователь заполняет форму (бизнес или блогер)
- Данные валидируются на клиенте
- Создается аккаунт в Firebase Authentication
- Сохраняется токен и данные пользователя в localStorage
- Пользователь перенаправляется на главную страницу

### Вход:
- Пользователь вводит email и пароль
- Firebase проверяет учетные данные
- Сохраняется токен (в localStorage или sessionStorage в зависимости от "Запам'ятати мене")
- Пользователь перенаправляется на главную страницу

### Состояние аутентификации:
- Firebase автоматически отслеживает состояние входа
- Пользователь остается залогиненным при перезагрузке страницы
- Токен автоматически обновляется при необходимости

## Файлы:

- `firebase-config.js` - Конфигурация Firebase
- `auth.js` - Логика аутентификации с Firebase
- `register.html` - Страница регистрации
- `login.html` - Страница входа

## Использование:

1. Откройте `register.html` для регистрации нового пользователя
2. Откройте `login.html` для входа существующего пользователя
3. Все данные сохраняются в Firebase Authentication

## Дополнительные возможности (опционально):

Если вы хотите сохранять дополнительные данные пользователя (имя, телефон, описание и т.д.), можно:

1. Включить Firestore в Firebase Console
2. Раскомментировать код в функции `registerUser()` в `auth.js` (строки с комментарием про Firestore)

## Проверка работы:

1. Откройте страницу регистрации
2. Заполните форму и зарегистрируйтесь
3. Проверьте Firebase Console → Authentication → Users - должен появиться новый пользователь
4. Войдите с созданными учетными данными

## Обработка ошибок:

Система обрабатывает следующие ошибки Firebase:
- Email уже используется
- Неверный формат email
- Слабый пароль
- Пользователь не найден
- Неверный пароль
- Проблемы с сетью
- И другие стандартные ошибки Firebase
